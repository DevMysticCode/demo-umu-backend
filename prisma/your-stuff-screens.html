Six screens · three modals · one design language

Buyer Profile · the confidence card
Sellers want to know they're talking to a real, ready buyer. The hero is a circular profile-strength gauge showing 92% with a "Trusted Buyer" label. Below, four signal cards (Identity / Funds / Chain / Mortgage) tell the story at a glance — the first three filled-teal "verified", the fourth still teal-paler waiting for the AIP letter. Detail rows below let the user dig in. The closing piece is a shareable digital business card in the deep teal-and-gold gilt language we've used for the passport books — it's what the user sends to a seller via the property page's "Message owner" flow.
Personal Information · the same sections, properly grouped
Your existing structure stays — Name & Contact, Address, Company, Comms Preferences, Identity Verification — but each one is now a single grouped card with rows separated by hairline dividers, instead of floating chips with edit pencils. Profile completion meter at the top (78%) gives the user a sense of "how done am I" without nagging. Empty rows like "Add company" use teal text in place of grey, so they read as opportunities not gaps. Verified row gets a teal-tinted highlight, with the Onfido date — they earned it, show it.
Collaborators · per-property, but warm
Each person card has a coloured-gradient avatar (different palette per role: solicitor deep teal, agent gold, partner coral, broker purple), name, role tag in tracked-out caps, organisation, and the properties they're attached to as small green location pills. Sarah covers all properties so hers is a quieter "All properties" pill; Michael covers two specific ones. Suggested empty state nudges the user: "Bring in a mortgage broker? They'll have direct access to your buyer profile." A floating teal + button is the way in. Tapping it opens the modal you see below — pick role, pick existing OR add new, attach to properties.
Saved Properties · life signals on each card
2-up grid of hearted properties with real illustrated SVG scenes per house — different sky colours, different facades, different garden details so they feel like real places not template thumbnails. Status pip lower-left tells you whether each one has a passport (deep-teal pill with gold text for "Passport ready", or quiet white pill for "No passport"). Top-left flag shows price moves: coral with a downward arrow for −£15k, gold with upward for +£8k. Hero stats line up top: 8 saved · 3 passport ready · 2 price changes · 5 viewed this week.
Calendar · everything in one place
Mini month grid up top with coloured dots beneath each day-number — teal for viewings, coral for compliance, gold for deadlines, purple for personal. Today is the navy filled circle. The 15th is selected (light teal ring). Below, an agenda grouped by day. Each event card has a coloured left border matching its type, time and duration on the left, title and meta in the middle, and a small tag pill. Pulls in viewings (from property bookings), compliance renewals (from your landlord passports), and deadlines (from your AIP, etc.) automatically — and lets the user add personal events with the floating + (modal shown below).
Your Documents · categories and recents
Search-first hero, then a 2x2 grid of colour-coded category cards (Identity teal-green, Property gold, Financial coral, Other purple — using the four-colour system that runs through the rest of the app). Each card shows a count. Below, a list of recently uploaded docs with realistic file-icon thumbnails (PDF coral, IMG green, DOC gold) shaped like a piece of paper with a folded corner. Floating upload button. Modal flow shows the upload experience: drag-drop zone (tinted teal), category picker, optional property link, expiry date — useful for time-sensitive docs like AIPs.
Modal pattern · centred, modern, contained
Three modal flows shown — Add Collaborator, New Event, Upload Document. All use the same pattern: blurred-out background of the screen behind, centred modal with a 24px corner radius, soft handle at the top, header with title and close × on the right. Body scrolls if needed. Footer pinned to the bottom with Cancel + a primary action. Form sections use 10px tracked-out caps as labels (consistent with the screens). Where there are multiple choices, they appear as radio buttons styled as cards with icons, not bare radio dots — much easier to read at a glance and matches the rest of the app.
What stays consistent across all six
Same nav bar pattern (back / centred title / right action). Same italic serif greeting. Same bold sans headline. Same personal stats line. Same floating action button on screens that need adding. Same colour vocabulary: teal for action, gold for value, coral for urgency, purple for personal/social. Same card depth and corner radii. The user moving from Buyer Profile to Calendar to Documents shouldn't need to learn anything — only what's in front of them.
